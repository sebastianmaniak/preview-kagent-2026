# Chat Channels

kagent can talk to your team on the chat platforms you already use. Each channel connects to your agents via the [A2A (Agent-to-Agent) protocol](/docs/kagent/examples/a2a-agents), so your agents respond directly in conversation threads.

Text is supported on every channel. Rich formatting, media attachments, and interactive elements vary by platform.

## How it works

Every chat channel follows the same pattern:

1. A lightweight bot or connector runs alongside your cluster (or externally)
2. Incoming messages are forwarded to kagent agents via the A2A protocol
3. Agents process the request using their configured tools and LLM
4. Responses flow back through the same channel to the user

Because channels use A2A, any agent you build works on any channel — no per-platform agent code required.

## Delivery notes

- **Slack** multi-person DMs route as group chats. Group policy, mention behavior, and session rules apply to MPIM conversations.
- **Telegram** replies that contain markdown image syntax are converted into media replies on the outbound path when the bot supports it.
- **MCP clients** connect directly to agent endpoints. No bot middleware is needed — the client speaks MCP natively.
